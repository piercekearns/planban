# Security Policy

Planban runs locally and can interact with project planning state through Codex,
CLI, local API, and MCP tools. Please avoid posting sensitive details publicly.

## Reporting a Vulnerability

If you believe you have found a security issue, please do not open a public GitHub
issue with private details.

Email:

```text
security@planban.ai
```

If that address is not available yet, contact Pierce Kearns directly and include
only the minimum information needed to start a private conversation.

## What to Include

Please include:

- affected Planban version or commit if known;
- install method;
- operating system;
- brief impact summary;
- reproduction steps with sensitive paths, board contents, logs, and credentials
  removed or redacted.

## Public Issues

Use public GitHub issues for ordinary bugs, docs feedback, feature requests, and
rough edges. Use this private security route for vulnerabilities, credential leaks,
unsafe update behavior, or issues that require sharing sensitive local context.
