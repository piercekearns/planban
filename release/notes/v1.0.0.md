# Planban v1.0.0

Planban is now available as a first public release.

Planban is a local, Codex-native Kanban board for agent-led planning. It gives you a roadmap board that lives on your machine, opens inside the Codex in-app browser, and gives Codex enough structured context to start from the right card, spec, plan, and next action.

The local core is open source under the MIT License. Feedback, focused pull requests, docs improvements, and reliability fixes are welcome through the public GitHub repository, with private security reports handled through `SECURITY.md`.

## Highlights

- Install Planban as a Git-backed Codex plugin marketplace from `piercekearns/planban`.
- Open the right local board from Codex with `/pb`, `/planban`, or natural prompts.
- Keep planning state local in `~/.planban`, with only lightweight repo discovery files in `.planban/`.
- Create, move, archive, restore, and delete roadmap cards through the local board.
- Duplicate whole boards into independent local planning spaces.
- Read and edit card specs and plans from the board.
- Use CLI, local API, and MCP surfaces so Codex can take structured local actions.
- Keep board view preferences local, including collapsed columns, hidden cards, archived visibility, and the Complete column's default hidden-card state.
- Use the built-in tutorial and feedback flow to learn the product, draft GitHub issues with your agent, and report rough edges without accidentally posting private local context.

## Install

Ask Codex:

```text
Install Planban from piercekearns/planban. Follow the Install With Codex details in the public GitHub README exactly, verify the plugin and MCP tools work, open the interactive tutorial in the Codex in-app browser, then ask whether I want to set up Planban for a local project.
```

Manual install instructions are in the README.

## Open Source

- Code is licensed under MIT.
- The Planban name, logo, wordmark, domain, social handles, and official release channels are covered by `TRADEMARKS.md`.
- Small, focused pull requests are welcome. Larger feature, install, update, API, MCP, or CLI contract changes should start with a GitHub issue.
- Security concerns should be reported privately using `SECURITY.md`, not through public issues.

## Notes For Preview Users

This release is intended to become the clean public starting point for the `piercekearns/planban` repository. If you used an older private or preview build, reinstall from this public repository or use Codex-guided update so your local marketplace points at the clean public release.
